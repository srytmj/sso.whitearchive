# BUG-002 — Middleware alias `scope` menunjuk ke class yang tidak exist di Passport 13

**Severity**: P0 — Blocker (GET /api/user dengan token valid selalu 500)
**Status**: Fixed ✅
**Found during**: Runtime verification session 2026-07-21

## Symptom

```
GET /api/user HTTP/500
{"message": "Target class [Laravel\Passport\Http\Middleware\CheckForAnyScope] does not exist."}
```

## Root Cause

`bootstrap/app.php` mendaftarkan alias `scope` ke `CheckForAnyScope` dan `scopes` ke `CheckScopes`. Di Passport 13.x, kedua class ini tidak ada — diganti dengan `CheckTokenForAnyScope`.

## Fix Applied

Update `bootstrap/app.php`:

```php
// Before
'scope'  => \Laravel\Passport\Http\Middleware\CheckForAnyScope::class,
'scopes' => \Laravel\Passport\Http\Middleware\CheckScopes::class,

// After
'scope'  => \Laravel\Passport\Http\Middleware\CheckTokenForAnyScope::class,
'scopes' => \Laravel\Passport\Http\Middleware\CheckTokenForAnyScope::class,
```

**File**: `bootstrap/app.php` — sudah difix di session ini.
