# BUG-003 — `RevokeTokenAction` memanggil method `refreshTokens()` yang tidak exist di Passport 13

**Severity**: P1 — Logout tidak merevoke refresh tokens (exception di-swallow)
**Status**: Fixed ✅
**Found during**: Runtime verification session 2026-07-21

## Symptom

Saat logout, `RevokeTokenAction::execute()` melempar:

```
BadMethodCallException: Call to undefined method Laravel\Passport\Token::refreshTokens()
```

Akibatnya, access tokens berhasil di-revoke tapi refresh tokens **tidak di-revoke** (exception muncul di tengah loop, iterasi berhenti).

## Root Cause

`RevokeTokenAction` memanggil `$token->refreshTokens()->update(...)` (plural). Di Passport 13.x, relasi ini adalah `refreshToken` (singular, `HasOne`), bukan `refreshTokens` (plural).

## Fix Applied

Update `app/Actions/Auth/RevokeTokenAction.php`:

```php
// Before
$token->refreshTokens()->update(['revoked' => true]);

// After
$token->refreshToken?->update(['revoked' => true]);
```

Menggunakan nullsafe operator karena tidak semua access token memiliki refresh token (misal: personal access tokens).

**File**: `app/Actions/Auth/RevokeTokenAction.php` — sudah difix di session ini.
